# Remove Manus Cloud / Non-Target Cloud Residue Plan

> Status: **Planning only — not implemented.** This document is the spec for a cleanup
> that a follow-up executor (Fable) will perform and push to production.

## Goal

The only production-facing external platforms for this app are:

- **GitHub / GitHub Pages** (frontend hosting + CI)
- **Supabase** (Auth, Postgres, Storage, Edge Functions = the production API)
- **Stripe** (payments)

Everything tied to **Manus Cloud** (the original scaffold vendor) and **Vercel** and
**Builder.io** is residue from the original template and must be removed **without**
disturbing the three target platforms, tRPC, or local Express dev.

## Risk Notes (traps)

1. **`vercel-build` is a name, not a Vercel dependency.** It runs `scripts/build.mjs`
   (pure Vite build → `dist/public`). GitHub Pages depends on it. Removing `vercel.json`
   is safe; renaming/removing the *script* is **not** part of this task and would break the
   Pages deploy unless the workflow is edited in lockstep.
2. **The debug collector is unreferenced yet still shipped.** Do not conclude "unreferenced =
   harmless." Vite's `publicDir` copy is what ships it. Verification must confirm it is
   **absent from a fresh `dist/public`** after removal.
3. **`dist/` and `.vercel/` are build output** (gitignored). Do not `git add` them; do not
   base conclusions on stale local `dist/` contents beyond confirming the copy behavior.
4. **Lockfile must be regenerated, not hand-edited.** After removing the Builder.io dep from
   `package.json`, run `pnpm install` so `pnpm-lock.yaml` updates cleanly.
5. **`DECISIONS.md` is history.** Prefer a "superseded" banner over deletion; erasing it
   removes useful context on why the stack changed.
6. **Unrelated uncommitted work is present.** `git status` shows modified
   `client/src/App.tsx`, `client/src/pages/Dashboard.tsx`, `client/src/pages/Documents.tsx`,
   `server/routers.ts`, `supabase/functions/_shared/routers.ts`, and untracked
   `client/src/pages/Profile.tsx`. **These are NOT part of this cleanup.** The cleanup commit
   must stage only residue-removal files (explicit `git add` of the paths below), leaving the
   product work uncommitted. Do not `git add -A` / `git add .`.

### Phase 1 — Remove active runtime Manus artifacts

- **Purpose**: Stop shipping the Manus debug collector to production.
- **Files likely touched**: `client/public/__manus__/debug-collector.js` (delete),
  `client/public/__manus__/` (remove dir).

  `404.html`, `.gitkeep`.
- **Rollback**: `git checkout -- client/public/__manus__/debug-collector.js` (pre-commit) or
  `git revert` the commit.

### Phase 2 — Remove Builder.io jsx-loc build plugin

- **Purpose**: Drop non-target build instrumentation.
- **Files likely touched**: `vite.config.ts`, `package.json`, `pnpm-lock.yaml`.
- **Actions**:
  - `vite.config.ts`: delete the `import { jsxLocPlugin } ...` line and remove
    `jsxLocPlugin()` from the `plugins` array → `const plugins = [react(), tailwindcss()];`
  - `package.json`: remove the `"@builder.io/vite-plugin-jsx-loc": "^0.1.1"` devDependency.
  - `corepack pnpm install` to regenerate `pnpm-lock.yaml` (removes the two `@builder.io/*`
    entries). Do not hand-edit the lockfile.
- **Verification**: `grep -rniE "builder\.io|jsx-loc" vite.config.ts package.json pnpm-lock.yaml`
  → expect none; `corepack pnpm run build` still succeeds.
- **Rollback**: `git checkout -- vite.config.ts package.json pnpm-lock.yaml`.

### Phase 3 — Remove stale Vercel deploy config (keep `vercel-build` script)

- **Purpose**: Eliminate the only non-target deploy target (Vercel), leaving GitHub Pages
  as the sole frontend deploy path.
- **Files likely touched**: `vercel.json` (delete), `vite.config.ts` (host allowlist).
- **Actions**:
  - `git rm vercel.json`.
  - `vite.config.ts`: change `allowedHosts: ["localhost", "127.0.0.1", ".vercel.app"]` to
    `allowedHosts: ["localhost", "127.0.0.1"]`.
  - **Do NOT** touch the `vercel-build` script or `scripts/build.mjs` (GitHub Pages uses it).
  - Optionally `rm -rf .vercel/` locally (untracked; not a git change).
- **Verification**: `deploy-frontend.yml` still references `pnpm run vercel-build`; that script
  still exists; `corepack pnpm run build` succeeds.
- **Rollback**: `git checkout -- vercel.json vite.config.ts` (restores deletion pre-commit) or
  `git revert`.

### Phase 4 — Clean docs to reflect the true stack

- **Rollback**: `git checkout -- DECISIONS.md STATUS.md`.
